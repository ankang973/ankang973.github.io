Java下载安装配置环境变量

无标签
发布日期:   2019-12-17

Java下载与配置

[toc]

  • 下载DK

要开始Java了，现在开始记下来我的收获，到此时间是2019年12月16日 17:19:52，天气有点阴沉有点冷，肉肉又复发流鼻涕。

1.废话不多说，直接开始，Java第一步就是要下载JDK。好的，第一缕疑惑出现了，我什么要下载JDK，这JDK是个啥？

答案：JDK是你使用Java中使用的工具。

就像你要当农民，你得有农具，你要当Java码农，你就要有JDK。农具有锄头镰刀，JDK有编译器，虚拟机。当你编写好你的Java源文件，通过JDK中的编译器进行语法检查，检查无误后生成一个二进制码类文件.class，这个新生成的二进制码类文件是给计算机看的，我是看不懂。

然后！使用JDK中的虚拟机运行刚刚新生的二进制码类文件，这时计算机开始按照你的命令做出你想要的结果，也就是你的代码发挥功效了，你的工作有体现了。

2.下载JDK

从Oracle官网下载JDK，据说现在多数使用JDK1.8版本，我也下的1.8版本，其实没太大，因为学的太浅，新版本的新特性轮不到你用。

JDK的下载是傻瓜下载，安装也是，有一点要注意在安装ＪＤＫ时注意第一个下一步页面，是否有有ＪＲＥ结尾的路径，这一步建议取消。取消后可以继续安装，如果不取消，你最后安装完成是有两个文件夹，除了你要的JDK1.8文件夹之外，他又给你安装了一个JRE文件夹，文件夹中的是虚拟机，而JDK1.8文件夹中已经有了虚拟机，所以是重复了。如果之前我说的你没有点击取消，现在有了JRE，可以直接删除，不影响编写和运行。

下载之后要配置环境

配置环境是为了能随时随地随心所欲地调用编译器javac.exe和虚拟机java.exe

配置方法一

依次点击：电脑右键，属性，高级系统设置，在高级栏里找到环境变量，点击环境变量，点击path变量，点击编辑，复制粘贴过来（不要手打，易出错）JDK文件夹的bin目录

如：D:\Program Files\Java\jdk1.8.0_231\bin将这个目录粘贴到path路径中所有路径之前，并在路径结尾处加上英语状态下的分号，这个英文分号，用以与其他路径隔离。（JDK安装时会安装在program files 文件夹下是因为你的64位系统，32系统就会在program files×86文件夹下）。

到这里还没有结束，继续点击完所用的确定， then win键+R输入cmd回车 重启DOS命令行 检查配置是否成功。可以在任意的非bin目录下输入Javac，java，如果使用出现相关的东西，那么恭喜你，环境配置成功了。 通过javac -version检查当前使用的javac版本。

配置方法二推荐

装逼的环境变量配置方法，步骤与方法一的相同。

打开path路径，在系统变量里新建名为JAVA_HOME，内容为D:\Program Files\Java\jdk1.8.0_231，（不包含bin目录）点击保存，然后在path路径中最前面加上%JAVA_HOME%\bin; (bin目录在这里），用两侧加%%动态获取变量，这样我们触动path路径的概率减少，出现错误的可能也随之减少，变量JAVA_HOME内任意位置可更改。

方法三临时配置方法

多用于不想改变原配置环境变量而暂时使用的方法。

*set *命令 查看、设置、删除Windows环境变量。

set path查看path环境变量。

set path=具体值，替换之前的环境变量，设定当前临时环境变量直至关闭本次DOS命令行。

set path=等号后空，清空path环境变量。

PATHEXT=.COM;.EXE;.BAT;.CMD;.VBS;.VBE;.JS;.JSE;.WSF;.WSH;.MSC这是path的扩展名，支持这些类型的文件，当你输入javac时不用加.exe即可启动javac.exe文件。

set path=新值;%path%在保留原环境变量基础上加上新值，%path%动态获取原path环境变量。

DOS命令行常见命令

*cd\ * 退回到当前根目录下

盘符名： 切换盘符

cls 清屏

exit 退出DOS
